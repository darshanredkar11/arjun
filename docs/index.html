🔧 How It Works

Step 1: Repository Analysis

Arjun scans your codebase and builds a dependency graph using Tree-Sitter (accurate symbol extraction for 6+ languages).

Step 2: Intelligent Ranking

Files are ranked using PageRank algorithm, considering:

  • How many files reference it (importance)
  • Number of symbols defined (value)
  • File size (ease of inclusion)
  • Special files (README, package.json)

Step 3: Content Detection

Arjun detects what type of content it is: Code, Protobuf, GraphQL, SQL, CSV, JSON, Logs, XML, YAML, Diff, Markdown, or plain text.

Step 4: Smart Compression

Based on content type, applies specialized compression:

Step 5: Context Budgeting

Fits compressed context into your token budget (4096, 8192, 200K, etc.). Greedily includes highest-ranked files first.

Step 6: One-Click Export

Copy formatted context to clipboard. Paste into Kiro. Done.

🚀 How to Use Arjun

Installation (5 minutes)

  1. Build extension: npm install && npm run esbuild
  2. Package: npm install -g @vscode/vsce && vsce package
  3. Install: code --install-extension arjun-0.1.0.vsix
  4. Restart VSCode

Workflow (3 steps)

  1. Click Arjun icon (⚡) in VSCode sidebar
  2. Run "Arjun: Analyze Repository" (Cmd+Shift+P → "analyze")
    Wait for completion (1-10 seconds)
  3. Run "Arjun: Build Optimized Context" (Cmd+Shift+P → "optimize")
    Stats panel opens showing token savings
  4. Click "Copy Context to Clipboard"
  5. Paste into Kiro and ask your question

Commands

Command What it does Shortcut
Analyze Repository Scan and index your codebase Cmd+Shift+P → "arjun analyze"
Build Optimized Context Generate compressed context for your prompt Cmd+Shift+P → "arjun optimize"
Compress Current File Show compression stats for active file Cmd+Shift+P → "arjun compress file"
Compress Logs Compress error logs with deduplication Cmd+Shift+P → "arjun compress logs"

🏗️ Architecture

System Design

VSCode Extension (TypeScript/Node) │ ├─ RepoAnalyzer │ ├─ Tree-Sitter Extractor (symbol extraction) │ ├─ PageRank Ranking (file importance) │ └─ Cache Manager (SQLite persistence) │ ├─ Compressor │ ├─ Content Detector (13 types) │ └─ Custom Compressors │ ├─ Code Compressor │ ├─ Log Compressor │ ├─ GraphQL Compressor │ ├─ Protobuf Compressor │ ├─ SQL Compressor │ ├─ CSV Compressor │ ├─ XML/YAML/Diff Compressors │ ├─ ContextBuilder │ ├─ Orchestrates analysis + compression │ └─ Fits context into token budget │ └─ UI ├─ Sidebar Tree View ├─ Stats WebView Panel └─ Commands

Data Flow

User opens project folder ↓ RepoAnalyzer.analyze() ├─ Scan files (with .gitignore awareness) ├─ Extract symbols (TreeSitter) ├─ Build dependency graph └─ Rank files (PageRank) ↓ Sidebar shows top 10 files ↓ User runs "Build Optimized Context" ↓ ContextBuilder.buildOptimized() ├─ Get prompt from editor ├─ For each top file: │ ├─ Read file │ ├─ Detect content type │ ├─ Compress (format-aware) │ └─ Add to context if fits budget ├─ Build final output └─ Display stats ↓ User clicks "Copy Context" ↓ Context copied to clipboard ↓ User pastes into Kiro ↓ Result: 50-70% fewer tokens!

❓ Frequently Asked Questions

How accurate is the token estimation?

Arjun uses ~4 characters per token (industry standard estimate). For exact counts, check Kiro's actual token usage in the API logs. Arjun's numbers are for comparison and relative savings.

Will my code be sent to the cloud?

No. Everything runs locally. No internet required. Your code never leaves your machine.

What if I want to use a different AI assistant (Claude, ChatGPT, etc.)?

The compressed context is plain text/JSON. Copy it into any assistant. Works with all of them.

Can I customize what gets compressed?

Future versions will support compression profiles. For now, Arjun auto-detects content type and applies best compression.

How fast is Arjun?

First run: 1-10 seconds (depends on repo size). Cached runs: 75% faster. Large repos (1000+ files) benefit most from caching.

What languages are supported?

Symbol extraction: TypeScript, JavaScript, Python, Java, Go, Rust, C/C++. Compression works on any file type.

Can I use Arjun with an offline Kiro instance?

Yes! Arjun is fully offline. Copy context from here, paste into your local Kiro instance. Perfect for air-gapped environments.

Is there a team/enterprise version?

Not yet. Roadmap includes: Git awareness, custom compression profiles, team metrics. Contact us for early access.

What about large files (10K+ lines)?

Arjun samples and compresses intelligently. For massive files, only relevant snippets are included based on ranking.

Can I trust the quality of compressed code?

Yes. Arjun preserves semantic meaning: imports, class definitions, method signatures, dependencies. Only removes comments and whitespace.
